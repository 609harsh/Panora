The Panora API uses uses two elements to indentify users: API keys and an id_linked_user.  
API Keys are used to authenticate yourself to our backend, while `id_linked_user` serves to identify which user's data you want to manipulate.
You can view and manage your API keys in the Panora Dashboard. Panora supports restricted API keys for granular permissions.
You can catch new `id_linked_user` and their corresponding platform by listening to webhooks.
Your API keys carry many privileges, so be sure to keep them secure! Do not share your secret API keys in publicly accessible areas such as GitHub, client-side code, and so forth.
